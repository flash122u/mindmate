# Step 3：对话质感（最高优先级）

## 目标

让"单纯聊天"就足够真实——不像 AI 客服，像微信里的朋友。

针对三个痛点对症下药：
- 回复不像真人 → 短消息分段 + 不秒回 + 口语化 prompt
- 连贯性差 → ✅ 已在 Step 1 修正（多轮对话上下文）
- 没有灵魂 → 丰富 SOUL + few-shot 示例 + mood 状态

## 任务清单

### ① 短消息分段（utils/splitter.py）
- 把 LLM 的一段回复智能拆成 2-3 条短消息
- 按句子/语气切分，模拟微信连发
- 每条之间有"打字延迟"（按字数估算）
- 通过 outbound 总线分别投递（链路已就绪）

### ② 不秒回 + 打字效果
- 收到消息后先等待随机思考时间（1-3s）
- 每条消息按长度模拟打字耗时
- 前端显示"对方正在输入..."
- prompt 强制：简短、口语、可以有语气词和不完美

### ③ 人格锚定
- 丰富 SOUL.md：说话习惯、口头禅、当前心情(mood)
- few-shot 示例对话注入 system prompt，锚定说话风格
- mood 状态影响当天语气（开心/疲惫/平静）

### ④ prompt 工程
- 重写 system prompt，强调"像真人朋友"
- 明确禁止：长篇大论、列点、过度礼貌、AI 腔
- 鼓励：短句、语气词、偶尔反问、有自己的情绪

## 验收标准

- 一句话回复会被拆成 2-3 条短消息连发
- 回复不秒回，有思考和打字延迟
- 回复简短口语化，不超过 2-3 句
- 同样问题在不同 mood 下语气不同
- 前端能看到分段消息逐条出现
