# 历史经验

> AI 与新成员写代码前先看这里，避免重复踩坑。

## 踩坑记录

### 1. SQLite 共享数据库 + 硬编码 ID → UNIQUE 约束冲突
- **问题**：多个测试共享 `memory/memory.db`，使用硬编码的 trace_id（如 `"test_trace_1"`）重复运行时触发 `sqlite3.IntegrityError: UNIQUE constraint failed: agent_traces.trace_id`
- **原因**：`MemoryStore()` 默认 workspace 指向同一个 `memory/` 目录，测试间无隔离
- **方案**：
  - 数据层：`add_agent_trace()` 使用 `INSERT OR REPLACE` 替代 `INSERT`
  - 测试层：trace_id 使用 `_uniq(prefix)` 辅助函数追加微秒时间戳
  - 长期建议：测试使用临时目录隔离 workspace

### 2. 规则路由的 identity 模式匹配过严
- **问题**：`test_router_identity_name` 断言 "你叫什么名字" 匹配 identity，但正则 `^(你是谁|你叫什么|你的名字|你.*谁)$` 要求精确匹配，"你叫什么名字" != "你叫什么"
- **原因**：正则模式写得太死，没有考虑自然语言的变体
- **方案**：将 identity 模式从 `re.match` 的精确正则改为 `re.search` 的子串匹配：`["你是谁", "你叫什么", "你的名字", ...]`，允许部分命中

### 3. AgentLoop 测试必须 mock LLM provider
- **问题**：测试中调用 `_process_message()` 会触发真实 DeepSeek API 调用，导致测试不稳定、慢、花钱
- **原因**：AgentLoop 在 `__init__` 中创建 `DeepSeekProvider`，走真实网络
- **方案**：测试中用 `loop.provider.chat = fake_chat` 替换为 async mock 函数。注意需要 mock 的是实例方法而非类方法

### 4. ruff F821 — 不能用 forward-reference 字符串做类型注解
- **问题**：`trace: "TraceBuilder" = None` 在 ruff 下报 F821 undefined name
- **原因**：ruff 不识别字符串形式的 forward reference
- **方案**：直接 import 类型并使用裸类型名：`trace: TraceBuilder | None = None`

### 5. 新增构造函数参数必须给默认值保持向后兼容
- **问题**：在 `AgentLoop.__init__` 和 `_build_context`/`_run_llm` 新增参数会影响所有现有调用方
- **原因**：85 个已有测试不传新参数会直接报 TypeError
- **方案**：所有新参数默认为 `None`（如 `trace: TraceBuilder | None = None`, `router: IntentRouter | None = None`），内部用 `if trace: trace.begin(...)` 守卫，对旧代码零影响

### 6. 依赖 LLM 的测试在 CI 中必然失败
- **问题**：`test_crisis.py`、`test_diary_dream.py`、`test_memory_consolidator.py`、`test_emotion_anchor.py` 等 8 个测试调用真实 API，没有 API key 或网络不通时失败
- **原因**：这些测试直接创建 LLM 相关的 manager 对象并发起真实请求
- **方案**：目前在本地跳过这些测试。长期应 mock LLM provider 或用 pytest.mark.skipif 条件跳过
