## 流程 2：主动问候推送 (P0)

```mermaid
sequenceDiagram
    participant PRL as ProactiveLoop
    participant ER as EnergyRegistry
    participant AL as AgentLoop
    participant LLM as DeepSeek API
    participant BUS as MessageBus

    loop 每 60 秒
        PRL->>PRL: asyncio.sleep(60)
        PRL->>ER: 遍历所有活跃用户
        ER-->>PRL: should_reach_out()?
        alt 能量模型允许
            PRL->>AL: generate_proactive(session_key)
            AL->>LLM: chat(结合上下文生成自然问候)
            LLM-->>AL: 自然问候消息
            AL->>BUS: publish_outbound(问候)
        else 冷却/静默/上限
            PRL->>PRL: skip, continue
        end
    end
```

### 能量模型决策维度
| 维度 | 控制参数 | 说明 |
| --- | --- | --- |
| 沉默时长 | `silence_threshold` | 用户多久没说话才适合主动 |
| 冷却时间 | `cooldown` | 两次主动推送的最小间隔 |
| 安静时段 | `quiet_hours` | 深夜/凌晨不打扰 |
| 每日上限 | `daily_limit` | 每天最多主动开口次数 |
