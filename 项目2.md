**名称**：MindMate・主动式 AI 心理陪伴 Agent 

**技术栈**：Python · asyncio · FastAPI · WebSocket · SQLite · MCP · Function Calling · DeepSeek(OpenAI SDK)

**项目简介**：一个具备拟人聊天风格、双层情绪记忆与主动关心能力的心理陪伴 Agent 平台。支持多用户状态隔离、分级风险预警，搭载意图路由与全链路可观测性，配套面向咨询师的六维可视化干预后台。

- 采用 asyncio 异步双循环架构分离被动问答与主动关怀，结合时间/沉默/每日上限四维能量模型实现拟人化主动触发。

- 设计多短句拆分与打字延迟推送机制，配合提示词模板 + few-shot 约束，高度还原真人即时通讯聊天体验。

- 构建双层情绪记忆体系，利用 LLM 提取"事件-情绪-触发线索-效价"锚点实现跨会话共情调取，并通过自动摘要压缩与时间衰减策略实现遗忘机制。

- 内置人格防御规则与动态关系层次推进策略，温和回避负面施压，保障 AI 陪伴边界的合理性与安全性。

- 设计分级心理风险检测规则（medium/high），精准识别高危情绪并自动触发预警与安抚话术，构建心理安全兜底机制。

- 搭载日记与梦境专属子 Agent，实现深度私密内容的场景化隔离与触发，提升陪伴角色的立体感与信任度。

- 自研规则优先意图路由器，risk/functional/identity 三类意图走预写模板零 LLM 调用，chat 意图进入完整 LLM 流程，Dashboard 实时展示路由分布。

- 自研 Agent 全链路可观测性系统（Run Trace），9 步记录每条消息的处理过程（INTENT_ROUTE→DEFENSE_CHECK→CRISIS_CHECK→ANCHOR_RECALL→SKILL_SELECT→LLM_CALL→TOOL_CALL→RELATIONSHIP_UPDATE→MEMORY_MAINTENANCE），每步带耗时与详情，数据持久化到 SQLite 并可在 Dashboard 查看。

- 构建上下文管理指标体系——窗口大小、压缩比、锚点召回数、System Prompt 字符数等 7 项量化指标——将 Agent 内部状态从"黑盒"变为可观测、可优化的工程数据。

- 借鉴 MindBridge 声明式 Skill 架构，自研陪伴场景包系统：6 个 SKILL.md（孤独/压力/自我怀疑/情感困扰/丧失/焦虑），YAML frontmatter 声明触发关键词，SkillLibrary 子串匹配动态选中后注入 system prompt——不改代码即可调整陪伴策略。

- 基于 FastAPI + WebSocket 构建实时双向通信，利用 chat_id 隔离多用户记忆/关系/能量/路由状态，保障会话独立性。

- 实现兼容 OpenAI 标准的多轮 Function Calling 循环（最多 4 轮），自研 4 个内置工具（天气查询/时间感知/活动推荐/危机检测）+ 自建 Music MCP Server（7 种心情 × 40+ 首精选歌单），通过 MCPManager 统一注册，工具来源对 LLM 透明。

- 开发面向咨询师的六维可视化后台（情绪趋势看板、风险预警列表、完整对话回放、Agent 运行轨迹、上下文管理指标、陪伴场景 Skill 状态），辅助专业心理咨询干预。

- 编写 150+ 单元测试（非 LLM 依赖全过），pytest + pytest-asyncio + ruff 保证代码规范。




**名称**：MindMate・主动式 AI 心理陪伴 Agent 

**技术栈**：Python · asyncio · FastAPI · WebSocket · DeepSeek(OpenAI SDK) · SQLite · MCP · Function Calling

**简介**：从零设计实现一个"伙伴型"而非"工具型"的主动式心理陪伴 Agent，主打真实感——模拟真人朋友的对话质感、情绪记忆与有分寸的主动关心，搭载全链路可观测性、意图路由与声明式陪伴 Skill 系统，并提供医生侧六维风险预警后台。


**拟人对话与主动架构**：单条回复按句拆分为多条短消息连发并模拟打字延迟，配合 prompt 风格约束 + few-shot 对抗"AI 腔"；被动响应与主动触发双循环经异步消息总线解耦，自研四维能量模型（沉默阈值/冷却/安静时段/每日上限）判断主动开口时机，由 LLM 结合上下文生成自然问候，规避模板化骚扰感。


**分层记忆系统**：客观日志（不可篡改）+ 主观体验（可整合/遗忘）双层架构；LLM 提取"事件—情绪—触发线索—效价"情绪锚点，按触发线索召回注入上下文实现"睹物思情"；超窗口对话压缩合并进 MEMORY.md，久远低关联锚点逐步弱化遗忘。


**意图路由与 Agent 可观测性**：规则优先三级意图路由器，risk/functional/identity 走预写模板零 LLM 调用，chat 走完整流程；自研 TraceBuilder 全链路插桩（9 步：路由→防御→危机→锚点→Skill→LLM→工具→关系→记忆维护），每步耗时/详情 SQLite 持久化 + Dashboard 可视化；上下文管理 7 项量化指标（窗口大小/压缩比/锚点召回/Skill 选中等）全部在后台实时可见。


**声明式陪伴 Skill 系统**：借鉴 MindBridge 架构，6 个 SKILL.md 文件（孤独/压力/自我怀疑/情感/丧失/焦虑），YAML frontmatter + 关键词列表驱动，SkillRegistry 文件系统发现 + SkillLibrary 子串匹配动态选中，注入 system prompt 引导小暖的场景化陪伴风格；不改 Python 代码即可由心理背景人员直接编辑策略文件，Dashboard 实时展示 Skill 加载状态。


**人格与安全**：雷区防御机制（按施压程度升级回避）赋予 Agent 心理边界；关系阶段随情感倾向自动演进；日记/造梦子 Agent 构建私密内在生活（默认隔离，仅交心时刻吐露）；分级危机检测规则命中即预警并注入关切指引。


**工具调用与 MCP 扩展**：实现 OpenAI 兼容多轮 Function Calling 循环（最多 4 轮）；自研 4 个内置工具——天气查询（wttr.in 免费 API）、时间感知（含深夜提醒）、治愈活动推荐（70+ 条 × 6 分类）、心理危机检测；自建 Music MCP Server（stdio 协议，7 种心情 × 40+ 首精选歌单），MCPManager 统一注册使工具来源对 LLM 透明。


**系统工程**：FastAPI + WebSocket 全双工通信，按 chat_id 实现多用户记忆/关系/能量/路由/工具状态完全隔离；MCPManager 启动时自动连接外部 server 并 list tools 注册；六维医生后台可视化情绪趋势/风险预警/对话回放/Agent 运行轨迹/上下文指标/Skill 状态；150+ 单元测试 + ruff 全通过。
